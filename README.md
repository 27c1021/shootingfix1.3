# ジャングルバルーンシューティング（ジャイロ操作版）

スマホを傾けて照準を動かし、画面のボタンで発射する
2画面プレイのバルーンシューティングゲームです。

- `index.html` … **パソコン側**（ゲーム画面）
- `smartphone.html` … **スマホ側**（コントローラー）

パソコンとスマホは、この後説明する「接続コード」を使って
Firebase Realtime Database経由でリアルタイムに通信します。


## 遊び方

1. パソコンで `index.html` を開きます（GitHub Pagesなど、
   httpsで配信されている必要があります）。
2. 画面中央に、4文字の**接続コード**とQRコードが表示されます。
3. スマホで `smartphone.html` を開き、QRコードを読み取るか
   （QRコードのリンクをタップ）、表示されたコードを手入力します。
4. スマホで「①ジャイロを有効にする」→（iPhoneの場合は
   モーションセンサーの許可ダイアログが出るので「許可」）。
5. スマホを傾けると、パソコン画面の照準（スコープ）が動きます。
   ずれを感じたら「②今の向きを中心に合わせる」でリセットできます。
6. パソコン側の「ゲームスタート」を押してカウントダウン後、
   スマホの「発射」ボタンで風船を撃ちます。


## 今回の書き直しで直したこと

### 1. 接続コード（部屋番号）をペアリングごとに発行するようにしました

以前のバージョンは、パソコンとスマホが常に同じ固定の部屋名
（`"main"`）を使ってFirebaseと通信していました。

もしこのページをGitHub Pagesのように**誰でもアクセスできる
場所**に置いていた場合、同時に別の人がこのゲームを開くと、
**全員が同じ部屋を取り合ってしまい**、照準が飛んだり、
発射が来なかったりする状態になります。これが
「スマホとパソコンがうまく接続できない」の大きな原因になっている
可能性が高いです。

今回の書き直しでは、パソコン側がゲーム画面を開くたびに
ランダムな4文字の接続コードを発行し、スマホはそのコードを
入力（またはQRコード読み取り）することで**1対1のペアだけが
つながる**ようにしました。複数のスマホ・パソコンが同時に
このゲームを開いても混線しません。

コードは同じタブ内でページを更新しても保持されます
（`sessionStorage`に保存）。別の相手とつなぎ直したいときは、
パソコン画面の「コードを作り直す」ボタンで新しいコードを
発行できます。

### 2. コードの重複を1か所にまとめました

Firebaseへの接続処理・エラーメッセージの整形処理が、
パソコン側 (`script.js`) とスマホ側 (`phone.js`) の
両方に別々に書かれていました。今回、共通処理を
`shared/firebase.js` と `shared/room-code.js` にまとめ、
どちらの画面も同じロジックを使うようにしました。
今後の修正や調整も1か所で済みます。

### 3. 効果音の取りこぼしを直しました

`sound/miss.mp3`（外れたときの音）が用意されていたのに、
どこからも再生されていませんでした。今回、風船に当たらなかった
ときにこの音が鳴るようにしました。

### 4. 接続エラー時に再接続できるようにしました

Firebaseへの接続に失敗した場合、以前は画面にエラー文言が
出るだけでした。今回は「再接続する」ボタンを追加し、
ページ全体を再読み込みしなくても再試行できるようにしました。

### 5. スマホが接続したら、自動でQRコード画面を消すようにしました

以前は、スマホがQRコードを読み取って「接続済み」になっても、
パソコン側の画面にはQRコード・接続コードがずっと表示されたまま
でした。

今回、スマホ側(`phone.js`)がFirebase接続に成功したタイミングで
`rooms/{roomId}/phoneConnected` に `true` を書き込むようにし、
パソコン側(`script.js`)はこれを監視して、スマホが接続した瞬間に
QRコード・接続コードの画面をふわっと消し、ゲーム画面を表示する
ようにしました（`onDisconnect()`を使っているので、スマホの
ブラウザを閉じたときも自動的に接続表示に戻ります）。

あわせて、接続待ちの間は風船やタイトルなどのゲーム画面が
QRコードの後ろに透けて見えて見にくかったため、接続が終わるまで
画面全体を単色の背景（`#connectionBackdrop`）で覆うようにしました。

### 6. スマホの「①ジャイロを有効にする」ボタンを直しました

iOSのSafariは、`DeviceOrientationEvent.requestPermission()`を
「click」イベントの中で直接呼び出さないと、信頼できるユーザー
操作として扱わず許可が下りないことがあります。以前は
「pointerdown」イベントで呼び出していたため、端末によっては
「ジャイロ：起動できませんでした」というエラーになっていました。
確実に許可が下りる「click」イベントに変更しました。


## 「接続できない」ときに確認してほしいこと（重要）

コードの書き直しだけでは直せない、**Firebaseプロジェクト側の
設定**が原因のことがよくあります。下記2点を
[Firebaseコンソール](https://console.firebase.google.com/)で
必ず確認してください（プロジェクト名: `wakutatu-shooting`）。

1. **Authentication → Sign-in method → 匿名（Anonymous）が
   有効になっているか**
   無効になっていると、パソコン・スマホどちらも
   「認証タイムアウト」や「接続エラー」で止まります。

2. **Realtime Database → ルール（Rules）** が、少なくとも
   下記のように `rooms` 以下を読み書きできる内容になっているか

   ```json
   {
     "rules": {
       "rooms": {
         "$roomId": {
           ".read": true,
           ".write": true
         }
       }
     }
   }
   ```

   ルールが `false` のままだと、認証は成功しても
   データの送受信だけが `permission-denied` エラーで
   失敗し、まさに「照準や発射が届かない」症状になります。

3. パソコン・スマホの両方が **https** で開かれているか
   （httpだとスマホのジャイロ許可ダイアログ自体が
   出せないブラウザがあります）。

4. パソコン画面右下の「コントローラー：」「ジャイロ照準：」の
   表示に、エラーメッセージがそのまま出るようにしてあります。
   接続できないときは、まずここに表示される文言を確認してください。


## ファイル構成

```
shooting_gyro_fixed/
├── index.html          … パソコン側の画面
├── smartphone.html     … スマホ側の画面
├── style.css            … パソコン側のスタイル
├── script.js             … パソコン側のロジック
├── phone.js              … スマホ側のロジック
├── shared/
│   ├── firebase.js       … Firebase接続の共通処理
│   └── room-code.js      … 接続コード（部屋番号）の共通処理
├── images/               … 背景・風船・照準の画像（そのまま）
└── sound/                … 効果音（そのまま）
```
